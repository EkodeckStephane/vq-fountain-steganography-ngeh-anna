# Promises-Evidence Matrix

Status: legacy ETHEGAN matrix. A new VQ-Fountain evidence matrix is required
after the synthetic and real-tokenizer experiments begin.

This matrix is the reviewer-facing guardrail for the pivot. No title, abstract,
or contribution should contain a claim stronger than the evidence status below.

| ID | Promise | Manuscript Location | Evidence Required | Evidence Available | Status | Action |

| P1 | The receiver does not need WYSAWIS-style cloud folders, location lists, or side metadata. | Title/Abstract/Contributions | Extraction from one stego image using only model weights, public packet parameters, and an optional shared seed/key; no original cover at receiver. | V3 embed/extract recovered a 1181-byte payload exactly from one stego image. V3 packet experiments also show exact clean-channel recovery on small Caltech101/COCO subsets. | Demonstrated on small subsets | Expand dataset size and document receiver inputs in method section. |
| P2 | The packet layer supports self-contained recovery from the stego image. | Method | Payload length, CRC32, ECC parameters, interleaving seed, and user bytes are represented in the image-carried packet rather than in cloud/folder/block-position metadata. | Implemented in `packet_v3.py`, `embed_payload_v3.py`, `extract_payload_v3.py`, and evaluated by `evaluate_v3_packet.py`. | Mechanism implemented | Present this as packet engineering tied to the WYSAWIS auxiliary-channel gap, not as a standalone novelty claim. |
| P3 | Capacity is reported as effective user payload, not nominal neural bitstream size. | Abstract/Contributions/Results | Tables separating nominal bpp, ECC overhead, packet overhead, effective user bpp, exact recovery rate, and failures. | V3 tables report 0.05, 0.10, and 0.25 nominal bpp with effective user bpp, exact recovery, and failures.
